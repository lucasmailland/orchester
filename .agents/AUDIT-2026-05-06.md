## 2. Bugs críticos (P0 — bloquean uso)

### 🔴 #1 — `/settings` rompe entera con TypeError [FIXEADO en este pase]

```
TypeError: Cannot read properties of null (reading 'toString')
  at fmt (components/settings/BillingSection.tsx:128:75)
```

**Causa:** `UsageBar` recibe `current` o `limit` con `null` desde la DB cuando el workspace no tiene `usage_events` del mes (auto-host fresh) y el plan es `enterprise` (limits = `Number.POSITIVE_INFINITY`). `null.toString()` explota.

**Fix aplicado** (commit pendiente):
```diff
- const fmt = (n: number) => (n >= 1000 ? `${(n / 1000).toFixed(1)}K` : n.toString());
- const limitTxt = isFinite(limit) ? fmt(limit) : "∞";
+ const fmt = (n: number | null | undefined) => {
+   const v = Number(n ?? 0);
+   return v >= 1000 ? `${(v / 1000).toFixed(1)}K` : String(v);
+ };
+ const limitTxt = limit != null && isFinite(limit) ? fmt(limit) : "∞";
```

Settings ahora carga.

## 3. Bugs altos (P1 — datos incorrectos)

### 🟠 #2 — `/employees` muestra cada usuario duplicado

Los 20 empleados del seed aparecen 2 veces cada uno (40 filas) tanto en `/employees` como en el contador del dashboard ("40 empleados activos"). Probablemente un JOIN sin `DISTINCT` en `db-queries.ts` (employee + workspace_member o similar).

**Acción:** revisar `getEmployees()` o equivalente. Esperado: 20 filas.

### 🟠 #3 — Dashboard: link "Ver todas las conversaciones" → 404

```html
<a href="/conversaciones">Ver todas las conversaciones</a>
```

Apunta a `/conversaciones` (español) pero la ruta es `/{locale}/conversations`. Hardcodeado en el componente del dashboard.

**Acción:** usar `useLocale()` y pathname dinámico: `/${locale}/conversations`.

### 🟠 #4 — Dashboard: KPIs con cálculos rotos

Visibles en el screenshot del dashboard:

- "Conversaciones / mes: **0** — `+−100%` vs mes ant., 160 el mes pasado" — formato `+-100%` es bug de prefijo (cuando el número es negativo no debería anteponer `+`).
- "Tokens este mes: **0**" pero "Costo estimado: **$11.24**" — incoherencia (si tokens=0, costo debería ser 0). Probablemente tokens=0 mide sólo el mes calendario actual mientras que costo es all-time.
- "Costo por agente: **+581%**" sobre el costo total — el delta no debería ir delante del valor absoluto principal.
- "Conversaciones hoy: **0** vs ayer" pero el seed tiene actividad — datos del seed están en abril, hoy es mayo, así que es coherente, pero la **UI no comunica esto** (debería decir "sin actividad hoy" en lugar de "0 vs ayer").

**Acción:** unificar todas las KPIs a la misma ventana temporal y arreglar el formateo de deltas.

### 🟠 #5 — Banner de provider falso positivo en `/agents`, `/flows`, `/knowledge`

Las tres pantallas muestran:

> ⚠ Aún no configuraste un proveedor de IA. Andá a Ajustes → Proveedores de IA…

Esto es **correcto técnicamente** (en `/settings` se ve "Anthropic: No configurado" para todos), pero **incongruente con el seed**: hay 6 agentes corriendo y 166 K tokens consumidos. La UI debería:
- Si **hay datos históricos** pero no hay key activa: mostrar "Tu proveedor está desconectado, los agentes no responderán a nuevas conversaciones. [Reconectar]"
- Si **no hay datos**: mostrar el banner actual.

**Acción:** chequear `hasAnyAgentMessage(workspaceId)` antes de mostrar el banner como "Aún no…".

### 🟠 #6 — `/conversations` lista todo como "Anónimo"

Cada item es `Anónimo · web · 7 mensajes · 26/4/2026, 1:45 p.m. · Cerrada`. El dashboard SÍ tiene los nombres reales (Patricia Domínguez, Diego Vargas, etc.) — la lista de conversaciones no resuelve el JOIN al employee. Tampoco se muestra el agente con el que conversó.

**Acción:** en `getConversationsForList()` hacer `LEFT JOIN employee ON conversation.employee_id = employee.id` y `LEFT JOIN agent`. Mostrar "Patricia Domínguez · con Sofia HR · 7 mensajes…".

### 🟠 #7 — `/settings` duplica secciones de "Miembros" y "API Keys"

Hay 2 bloques distintos para lo mismo:

- "Miembros e invitaciones" (form de invite vacío) + "Miembros del Equipo" (tabla con Demo Admin · Owner) — son la misma feature partida en 2 componentes.
- "API Keys" (lista vacía + create form) + "Claves API" (con datos hardcodeados "Production Key", "sk-orc-p••••••••", "Last used 2 hours ago") — el segundo bloque parece un mock viejo que sobrevivió.

**Acción:** eliminar los duplicados mock; unificar la sección.

### 🟠 #8 — `/settings`: "Gestionar suscripción" se muestra en self-host

Plan dice "Enterprise · $0/mes" (correcto, vino de `isSelfHosted()`) pero el botón "Gestionar suscripción" sigue ahí — clickearlo abriría el portal de Stripe que NO existe en self-host.

**Acción:** envolver el botón en `{isStripeEnabled() && <button>...}`.

## 4. Issues medios (P2 — UX / accesibilidad / pulido)

### UX

1. **AgentStudio:** botón "Sugerencias iniciales" tiene rol `button` cuando es un label de sección (uid=5_41 en el snapshot).
2. **Dashboard:** la tabla "RENDIMIENTO DE AGENTES · 6 AGENTES" muestra 6, pero el header de KPIs dice "8 activos · de 9 totales" — discrepancia. Probablemente uno filtra DRAFT y otro no; los counts tienen que estar alineados.
3. **`/conversations`:** sin paginación visible, scroll infinito sin indicador. Para workspaces con miles de conversaciones esto es problemático.
4. **AgentStudio test chat:** el botón send está deshabilitado correctamente, pero falta atajo Enter para enviar, Shift+Enter para nueva línea.
5. **`/integrations`:** todas las integraciones dicen "PRÓXIMAMENTE" + botón "Notificarme" deshabilitado. Es honest pero deja la sensación de que no hay roadmap. Sugerencia: mostrar un timeline ("Q3", "Q4") o un link a roadmap real.

### Accesibilidad

6. Botones sin texto/aria-label en varios lugares (lista de agentes, color picker del AgentStudio, toggles de notificaciones, copiar/eliminar API keys, paginación de "Activar modo presentación"). DevTools issue panel reportó: "**No label associated with a form field (count: 2)**" y "**A form field element should have an id or name attribute (count: 11)**".
7. Toggle de "Notificaciones" sin estado leído por screen reader (button vacío).
8. Color picker de AgentStudio: 8 botones sin texto identificando el color → screen reader inutilizable.

### Sidebar / nav

9. La entrada "Empleados" en el menú: la decisión previa fue "el organigrama solo muestra agentes y squads". Pero `/employees` sigue siendo una sección de primer nivel. Decidir: ¿se queda como directorio HR o se elimina? Si se queda, no debería ser "MAIN" sino "DATA" o moverla bajo `/settings`.
10. "MAIN" + "SYSTEM" como agrupadores: SYSTEM tiene `Canales`, `Integraciones`, `Configuración` — Canales más bien es feature operativa, no system. Sugerencia: `MAIN` (Inicio, Equipos, Organigrama, Agentes, Flujos, Conocimiento, Conversaciones), `OPERACIONES` (Canales, Integraciones), `WORKSPACE` (Empleados, Configuración).

### Branding / look

11. Header con "LIVE" + dot rojo + "Activar modo presentación" — el "LIVE" no se distingue del status indicator y confunde. Renombrar "ENTORNO LIVE" o quitarlo en self-host.
12. Avatar de usuario muestra "DA" sin label `Demo Admin` al hover.
