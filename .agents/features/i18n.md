# i18n

**Files:**
- `apps/web/i18n/{routing,request}.ts`
- `apps/web/messages/{es,en,pt-BR}.json`
- `apps/web/middleware.ts` (locale negotiation)

**Owner:** localization
**Status:** stable; coverage incomplete (many hardcoded strings)

## Purpose
Trilingual UI (Spanish, English, Brazilian Portuguese) with locale-prefixed
URLs and per-namespace translation keys.

## Planning (initial design)

### Routing
- All locale-aware routes live under `/[locale]/...`.
- Default locale: `es`.
- `middleware.ts` redirects `/` → `/es` based on `Accept-Language` and
  cookie.

### File structure
- `messages/<locale>.json` — single JSON per locale, namespaces nested.
- Server: `await getTranslations({ locale, namespace })`.
- Client: `useTranslations('namespace')`.

### Decisions & trade-offs
- **3 languages chosen** based on target market (LatAm + US + Brazil).
- **Hardcoded strings tolerated** during rapid development. Phase 8 sweep
  to fully translate.
